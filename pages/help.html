What is this?

This app can read and write to a GNUCash MySQL database using a PHP script I wrote (explained below).

The PHP script needs to be accessible over the internet and the script needs to be able to access the MySQL database that you want to use (usually localhost).

I have the PHP script running on a debian-based VPS with the MySQL server accessed through a VPN (OpenVPN), works great!

So where is this PHP script?
Tap here to copy the PHP script to your device so you can copy it to your web server.

Updates to the PHP script are packaged with updates to this app. The app will let you know when to update the PHP script.

I don’t understand this accounting voodoo...

In accounting, money comes from one account and goes to another. For example, when you get paid, money comes from an income account and goes to your bank account

Likewise, when you buy something, money comes from your credit card/bank and goes to an expense account, like dining.

Reconciling

Once transactions are posted/cleared (not pending), compare the amount that your statement shows for the transaction with the amount you entered into GNUCash. If you need to adjust the amount, do so. Once everything matches, reconcile it. If the dates are off by more than a few days, you should adjust the date as well. It helps in the future if you need to lookup a transaction.

In the View Account page, red transactions are not reconciled. In the Overview page, accounts in red have unreconciled transactions.
